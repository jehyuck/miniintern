# 기능흐름도

### 유저

- 회원가입 흐름

1. 사용자가 회원가입 버튼 클릭
2. 아이디 입력
   - 이미 존재하는 경우 → 에러 메시지 반환 ("이미 사용 중인 아이디입니다.")
3. 이메일 입력
   - 이메일 인증 코드 발송
   - 사용자가 인증 코드 입력
     - 인증 실패 (코드 불일치 / 만료) → 에러 메시지
4. 비밀번호 입력
5. 관심사 입력 (선택)
6. 모든 입력이 유효하면 회원 정보 저장
7. 성공 응답 반환 (userId)

- 로그인
- 유저 정보 수정
- 유저 삭제
- 호스트 등록

### M클래스

- M클래스 생성

- M클래스 상세 조회
- M클래스 삭제
